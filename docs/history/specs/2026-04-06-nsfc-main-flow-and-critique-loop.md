## 目标

冻结 `Med Auto Grant` 第一条可执行主线，也就是医学场景下 `NSFC` 通用申请骨架的主流程，以及围绕“导师批注 -> 修订 -> 复审”的 critique / revision loop。

这个文档回答的不是“系统里有哪些对象”，而是：

- 国自然申请主流程到底怎么走
- `Auto` 与 `Human-in-the-loop` 在哪些节点分叉
- 什么情况下允许继续
- 什么情况下必须停下来补证据、改题或重写

## 总体原则

第一版主流程必须服从以下判断顺序：

1. 先判断科学问题是否成立
2. 再判断申请人与问题是否适配
3. 再判断技术路线是否支撑问题
4. 最后才讨论文字是否漂亮、格式是否完整

也就是说，流程不能被“先出一版完整草稿再说”绑架。

## 主流程

### Phase 1. 输入接入与申请画像整理

输入包括：

- 申请人简历 / bio
- 代表性成果
- 在研项目
- 预实验结果或预实验缺口
- 候选方向 / 题目
- 申请要求与章节框架

目标输出：

- 一份结构化的申请人画像
- 一份前期基础与证据清单
- 一份当前可用材料缺口表

Gate：

- 资料是否足以判断申请人已有积累
- 是否已经知道哪些关键证据缺失

### Phase 2. 方向候选生成与收敛

系统需要从已有积累和候选主题中收敛出少量可申请方向，而不是一开始就写正文。

目标输出：

- 2 到 5 个方向候选
- 每个方向的必要性判断
- 每个方向的主要风险与证据缺口

Gate：

- 方向是否与申请人已有积累相连
- 是否能在医学语境下说清理论或机制缺口

### Phase 3. 科学问题提纯

这是整条主线的核心阶段。

系统必须区分：

- 科学问题
- 工程任务
- 临床需求

目标输出：

- 一条主科学问题
- 0 到 2 条支撑性子问题
- 对应的知识边界、未知机制、突破视角与可证伪表述

停止条件：

- 如果只能说出“做一个系统”
- 如果只能说出“提高准确率”
- 如果只能说出“解决某个临床痛点”

出现以上情况时，流程必须回退到方向阶段继续提纯，不能硬写。

### Phase 4. 立项依据与必要性链条构建

这一阶段要从背景、现状、缺口、科学问题一路推到“为什么必须做这个研究”。

目标输出：

- 一条完整的立项依据论证链
- “为什么是这个问题，而不是别的问题”
- “为什么需要这套方案，而不是任意方案”

Gate：

- 是否能解释不做这项研究时，该领域到底缺了什么
- 是否能说明该 proposal 不是任意替换的装饰路线

### Phase 5. 申请人与课题适配度论证

这一阶段解决的是“为什么该申请人来做”。

目标输出：

- 前期基础与科学问题之间的证据链
- 申请人独特性的陈述
- 强相关成果、预实验与方法栈映射

停止条件：

- 如果只能泛泛罗列简历
- 如果前期基础和当前问题明显脱节
- 如果存在强行蹭热点痕迹

则不能进入正文成稿阶段。

### Phase 6. 申请书提纲生成

只有在科学问题、必要性链条和申请人适配度都足够清楚后，才允许生成提纲。

目标输出：

- 申请书总提纲
- 每个章节的核心论点
- 每节需要调用的证据

Gate：

- 提纲是否围绕科学问题展开，而不是围绕模板字段展开
- 章节之间是否共享同一条论证主线

### Phase 7. 正文草稿生成

目标输出：

- 关键章节草稿
- 章节级证据映射
- 当前草稿版本记录

要求：

- 草稿必须忠实继承前面冻结的问题与链条
- 不允许在写作阶段悄悄替换核心科学问题

### Phase 8. 导师批注

这一阶段输出“导师批注”式 critique。

第一版批注固定围绕：

- `60%` 必要性与科学价值
- `30%` 胜任力与独特性
- `10%` 可行性

每轮批注至少要给出：

- 一针见血的总体诊断
- 当前识别出的科学问题
- 为什么它不够科学，或哪里仍工程化
- 建议修改版
- 立项依据链条修补建议
- 申请人与研究适配度建议

### Phase 9. 修订计划与执行

批注不能停留在意见列表，必须收束为修订计划。

目标输出：

- 修订优先级
- 对应动作项
- 需要补充的证据
- 需要重写的段落
- 修订后复审的重点

## 导师批注闭环

第一版 critique / revision loop 固定为：

1. 生成当前草稿的导师批注
2. 把批注拆成结构化修订任务
3. 执行修订
4. 比较修订前后版本
5. 重新批注
6. 判断是否继续迭代或停止

### 闭环停止条件

满足以下条件时可以停止：

- 科学问题表述稳定
- 立项依据链条没有明显断裂
- 申请人适配度证据足够
- 草稿剩余问题主要是局部表达或格式整理

### 闭环强制回退条件

满足以下任一条件时，必须回退而不是继续局部润色：

- 批注指出核心科学问题不成立
- 立项依据无法支撑问题必要性
- 申请人前期基础与问题明显错位
- 关键证据缺口无法通过表述技巧掩盖

## Auto 与 Human-in-the-loop 的分工

### Auto

`Auto` 模式下，Agent 默认自动推进以下动作：

- 输入整理
- 方向候选归并
- 科学问题初稿生成
- 立项依据链条搭建
- 提纲生成
- 草稿生成
- 导师批注
- 修订计划生成
- 多轮修订

### Human-in-the-loop

`Human-in-the-loop` 模式下，以下节点默认要求人确认：

- 方向冻结
- 主科学问题冻结
- 立项依据主链冻结
- 提纲冻结
- 正式送审前版本冻结

Agent 继续负责：

- 调研
- 证据整理
- 预实验需求清单
- 草稿改写
- 批注归并
- 修订追踪

## 第一版最重要的失败模式

系统必须优先防止以下失败模式：

- 把工程目标伪装成科学问题
- 立项依据只是在堆背景，没有推到知识边界
- 申请人适配度只是在堆简历，没有证明“为什么是你”
- 提纲与正文脱离已经冻结的科学问题
- 批注停留在措辞层，没有逼近逻辑硬伤
- 修订只做润色，不做结构性返工
