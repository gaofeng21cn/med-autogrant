## MVP 目标

第一阶段 MVP 聚焦医学场景下的 `NSFC` 通用申请骨架，而不是先绑定某一个高度特化的项目类型。

这一版系统优先解决的是：

- 科学问题提纯
- 必要性与科学价值论证
- 申请人与问题的适配度
- 申请书结构与草稿生成
- 导师式批注与修订闭环

## 评审内核

第一版批注内核先冻结为：

- `60%`：必要性与科学价值
- `30%`：申请人胜任力与独特性
- `10%`：可行性

其中最高优先级要求是：严格区分真正的科学问题与以下对象：

- 工程任务
- 临床需求陈述
- 泛泛的准确率提升目标

## MVP 输入

- 申请人简历或 bio
- 代表性成果
- 在研项目
- 可复用证据与预实验结果
- 候选方向或题目
- `NSFC` 申请要求与章节结构

## MVP 输出

- 方向收敛建议
- 凝练后的科学问题表述
- 必要性 / 科学价值论证链
- 申请人与课题的适配度说明
- 申请书关键章节草稿
- 导师式批注
- 修订计划与证据缺口清单

## 非目标

第一阶段 MVP 不尝试：

- 覆盖所有基金体系
- 变成 reviewer-owned 的基金评审平台
- 一开始就优化所有项目类型的模板差异
- 取代医学 `Research Ops`
